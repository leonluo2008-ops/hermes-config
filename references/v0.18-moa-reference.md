# Hermes v0.18 MoA 实战参考(聚鑫 + minimax + zai)

> 实战日期: 2026-07-08 · 升级 v0.16.0 → v0.18.1 后配置 MoA(Mixture-of-Agents)。
> 配套阅读: `SKILL.md` 4.6a 节(MoA 4 字段陷阱) + 4.6b 节(config 保护)。

## 1. 真实 schema(从源码验证,不是 changelog 推的)

**provider 合法字段**(`hermes_cli/config.py:4607-4617` + `agent/agent_init.py:423`):
```yaml
providers:
  <name>:
    base_url: <string>           # 必填
    api_key: <string>            # 可选,字面量,不推荐
    key_env: <env_var_name>      # 推荐,从 .env 读
    api_mode: <enum>             # 必填
    # 合法值: chat_completions | anthropic_messages | codex_responses
    #         | bedrock_converse | codex_app_server
    # ❌ openai_chat 不存在
    model: <string>              # 默认模型
    context_length: <int>        # 上下文长度
    extra_headers: <dict>        # 可选
    request_timeout_seconds: <int>
    stale_timeout_seconds: <int>
```

**MoA preset 合法字段**(`hermes_cli/moa_config.py:108-148`):
```yaml
moa:
  default_preset: <string>                       # 选一个 preset 名字
  presets:
    <preset_name>:
      aggregator: {provider: <name>, model: <string>}
      reference_models:                          # ✅ 正确字段名
        - {provider: <name>, model: <string>}
      reference_temperature: <float>             # 可选
      aggregator_temperature: <float>            # 可选
      max_tokens: <int>                          # 可选
      reference_max_tokens: <int>                # 可选
      fanout: <per_iteration | user_turn>        # 可选
      enabled: <bool>                            # 在 preset 内,不是顶层
```

## 2. 4 字段陷阱(最易踩)

| 想写 | 实际 | 错的后果 | 检测 |
|---|---|---|---|
| `env_var: KEY` | `key_env: KEY` | env_var 被静默忽略,key 不映射 | `grep env_var providers` 应该有 0 命中(全改成 key_env) |
| `api_mode: openai_chat` | `api_mode: chat_completions` | 启动报 schema 错 | 直接改 |
| `references: [...]` | `reference_models: [...]` | MoA 静默回退到默认 preset,你的 advisor 全丢 | 跑一次 `/moa` 看输出有没有"using default" 提示 |
| `moa.enabled: true` | **删除** | 顶层 `moa.enabled` 不被读,会让人误以为没启用 | grep `moa.enabled` 应只在 preset 内有 |

**额外必读**:
- `learn_enabled` 整个 v0.18 不存在 —— `/learn` 是**手动 slash command**
- 真"自动学习"走 `curator.enabled` + `curator.interval_hours`
- `verify_on_stop: true` 是**全局**,会让飞书消息也跑 build 验证 → huiben 内容创作被卡
  - 想要 coding 开、messaging 关 → 留 `"auto"`(默认行为)
  - 想要 coding 也开、`huiben` 不开 → 只在 default profile 加 `true`

## 3. 聚鑫中转接入样板(本次实战)

**base_url**(用户 2026-07-08 确认): `https://api.jxincm.cn`
**接口模式**: OpenAI Chat Completions 兼容(`POST /v1/chat/completions` + Bearer token)
**模型名** 以聚鑫后台"支持模型"列表为准(可能与 Google 官方版本号不同)

```yaml
# ~/.hermes/config.yaml — 加在 providers: 段
providers:
  juxin-gemini:
    base_url: https://api.jxincm.cn/v1
    api_mode: chat_completions
    key_env: JUXIN_API_KEY
  juxin-gpt:
    base_url: https://api.jxincm.cn/v1
    api_mode: chat_completions
    key_env: JUXIN_API_KEY

# 加在文件末尾(任意位置,hermes 合并)
moa:
  default_preset: my-council
  presets:
    my-council:
      aggregator:
        provider: juxin-gpt
        model: gpt-4o
      reference_models:
        - provider: minimax-cn
          model: MiniMax-M3
        - provider: zai
          model: glm-5.2
        - provider: juxin-gemini
          model: gemini-2.5-flash    # 实际模型名以聚鑫后台为准
```

```bash
# ~/.hermes/.env — 加一行
JUXIN_API_KEY=<从聚鑫后台"令牌"页复制,51+ 位>
```

## 4. 触发 MoA(2 选 1)

```bash
# A. 会话级触发(不改 model.default,推荐)
hermes chat --provider moa --model my-council

# B. 永久改 model.default(注意:日常所有任务都走 MoA,token 钱翻 4 倍)
# 在 model: 段改:
#   model:
#     default: my-council
#     provider: moa
```

**推荐**: A 方案,日常默认单模型,需要时 `--provider moa` 切。

## 5. 验证清单(改完必跑)

```bash
# 1. config 合法
hermes config check

# 2. provider 探测
hermes model probe juxin-gpt           # 应返回模型列表
hermes model probe juxin-gemini

# 3. 单调用测试
hermes chat --provider juxin-gpt --model gpt-4o "say hi"

# 4. MoA 触发测试
hermes chat --provider moa --model my-council "1+1=?"

# 5. 完整 doctor
hermes doctor                           # ⚠️ 实测可能 3-5 分钟才返回
```

**实测坑**: `hermes doctor` 在 v0.18 可能跑 3-5 分钟才返回(检查 N 个 provider + MCP + gateway),**别中途 kill**。

## 6. 回退链

```bash
# 改前必做
cp ~/.hermes/config.yaml ~/.hermes/config.yaml.bak.<reason>-$(date +%Y%m%d_%H%M%S)

# 回退 1: 从最近 .bak
ls -lt ~/.hermes/config.yaml.bak.* | head -3
cp ~/.hermes/config.yaml.bak.<最新> ~/.hermes/config.yaml

# 回退 2: hermes 自带 export(更稳)
hermes config export > ~/.hermes/config.yaml.full-backup-$(date +%Y%m%d_%H%M%S)

# 重启 gateway 让新 config 生效
systemctl --user restart hermes-gateway
```

⚠️ **`~/.hermes/` 不是 git 仓库**,`git checkout` 救不了 config。`~/.hermes/hermes-agent/` 才是 git 仓(装代码用)。

## 7. 模型名核对(常见坑)

**Gemini**: Google 官方没 `gemini-3.5-flash` 这版本,聚鑫可能自命名。聚鑫后台"支持模型"列表第一列 = 真实模型名,直接抄:
- `gemini-2.5-flash` —— ✅ Google 官方(便宜快速)
- `gemini-2.5-pro` —— ✅ Google 官方(贵强)

**GPT**:
- `gpt-4o` —— ✅ 稳定主力
- `gpt-4-turbo` —— 老版本
- `gpt-5` / `gpt-5-codex` —— 较新,聚鑫可能没接入或分组限速
- 同样以聚鑫后台为准
